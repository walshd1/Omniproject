# proof · defs

JSON definitions owned by the **proof** module — screens, primitives, artifacts, and mappings that
belong to deliverable review/sign-off — live here, co-located with the code that renders them. The
folder name mirrors the module name so a def's home is unambiguous from its path alone.

The proof annotation itself is a shipped primitive seeded into the backend catalogue, so this folder
is currently a convention anchor: any proof-specific screen or mapping def added later belongs here
rather than in the shared `src/definitions/` tree.
